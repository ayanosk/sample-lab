# 業務終了ログ作成

今日の作業内容を記録し、明日の研究を円滑に再開するための引き継ぎ資料を作成する。

**今日の完了事項:** $ARGUMENTS

## 手順

### 1. 今日のログファイルを確認する

```bash
TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M JST (%a)'
```

今日のログファイル `lab/logs/YYYY-MM-DD.md` を Read で読む（存在する場合）。

### 2. git差分から今日の作業内容を自動取得する（必須・省略不可）

```bash
git log --all --after="YYYY-MM-DDT00:00:00+09:00" --before="YYYY-MM-DDT23:59:59+09:00" --stat --format="%h %s"
```

コミットメッセージとファイル変更から完了タスクを復元する。

**重要**: git差分から判断できることをユーザーに質問してはならない。

### 3. ログを作成する

Read ツールで `assets/log_template.md` を読み、この形式で業務終了ログを生成する。
既存ログがある場合は末尾に `## 業務終了ログ` セクションを追記する。

### 4. ユーザー確認ステップ（必須）

git差分から把握済みの内容を提示し、「**上記以外に、gitに残らない作業（対面の打ち合わせ・手書きメモ・電話・物理的作業など）はありますか？**」と確認する。追加事項があればログに反映してから保存する。

### 5. エージェント日次ログの確認（毎日・必須）

Read ツールで `references/agent_daily_log.md` を読み、この手順でエージェント日次ログを確認・保存する。

### 6. 金曜日の追加手順（今日が金曜日の場合のみ）

Read ツールで `references/friday_routine.md` を読み、この手順でハートビートを実行する。

## ログ管理

- 保存場所: `lab/logs/YYYY-MM-DD.md`
- 朝の `/daily-start` が前回のログを自動的に読み込む
