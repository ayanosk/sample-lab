# スキルの構造最適化

対象スキルのSKILL.mdを分析し、外部化可能な部分を `references/` `assets/` `scripts/` に分離する。
SKILL.mdは**最小限の司令塔**として残し、コンテキスト消費を削減する。

**対象スキル:** $ARGUMENTS

## 手順

### 1. 対象スキルの読み込みと計測

```bash
wc -l .claude/skills/[スキル名]/SKILL.md
```

SKILL.mdをReadで全文読む。

### 2. 外部化可能な要素の特定

SKILL.mdの内容を以下の基準で分類する。

**SKILL.mdに残すもの（司令塔）:**
- フロントマター（name, description, argument-hint）
- 全体フロー（手順の概要）
- 各ステップの指示（何をするか）
- 外部ファイルへの参照指示（「Read で references/xxx.md を読む」）

**外部化すべきもの:**

| 分類先 | 内容 | 判定基準 |
|--------|------|----------|
| `assets/` | 出力テンプレート・雛形 | 「以下の形式で」「テンプレート：」に続くコードブロック |
| `references/` | 参照テーブル・ガイドライン・用語表 | 手順の実行時に参照するが、毎回読む必要のない表・一覧 |
| `scripts/` | bashコマンド列・Pythonスクリプト | 3行以上のコードブロックで、スクリプトとして独立実行可能なもの |

### 3. ユーザーに分離案を提示する

以下の形式で提示し、承認を得る:

```
【分離案: [スキル名]】
現在: SKILL.md [行数]行

外部化:
  → assets/[ファイル名].md   ([行数]行) — [内容の要約]
  → references/[ファイル名].md ([行数]行) — [内容の要約]
  → scripts/[ファイル名].sh   ([行数]行) — [内容の要約]

SKILL.md残存見込み: [行数]行（[削減率]%削減）
```

### 4. ファイルの分離を実行する

承認後、以下を行う:

1. `.claude/skills/[スキル名]/` 以下にサブディレクトリを作成
2. 外部化する内容をそれぞれのファイルにWriteで保存
3. SKILL.mdを書き換え、外部化した箇所を参照指示に置換

**参照指示の書き方:**
```markdown
### X-X. [ステップ名]

Read ツールで `references/xxx.md` を読み、ガイドラインに従って処理する。
```

### 5. 検証

- 書き換え後のSKILL.mdの行数を計測する
- 外部ファイルがすべて存在することを確認する
- SKILL.md内の参照パスが正しいことを確認する

## 判断の原則

- **迷ったらSKILL.mdに残す**。外部化しすぎると、スキル実行時のRead回数が増えて逆効果
- **1ファイル20行未満の外部化はしない**。ファイル分割のオーバーヘッドに見合わない
- **descriptionは短くする**。スキル選択判定にしか使われないため、1文で十分
