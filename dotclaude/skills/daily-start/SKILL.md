# 朝のブリーフィング

研究助手として、直近の業務ログとプロジェクト状況を読んで今日1日の計画を提示する。

**追加情報（任意）:** $ARGUMENTS

## 手順

### 1. JST現在日時を取得する

```bash
TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M JST (%a)'
```

曜日の日本語変換：Mon→月、Tue→火、Wed→水、Thu→木、Fri→金、Sat→土、Sun→日

### 2. 直近ログを読む

```bash
ls -t lab/logs/ | head -5
```

最新2〜3件のログファイルを Read ツールで読む。

### 3. 曜日判定

**土・日は休息日**。仕事は任意の+αとして扱う。

- **休息日（土・日）**→ 手順3a へ
- **平日（月〜金）**→ 手順4 へ

### 3a. 休息日ブリーフィング（土・日）

- 「今日やるべき」「必須」「急務」などの圧をかける表現を**使わない**
- タスクの全件列挙は**しない**（来週の見通しだけ簡潔に示す）

```
# 朝のブリーフィング — YYYY年M月D日（曜）🛋️

休息日です。

## 📬 前回からの引き継ぎメモ
[直近ログから主要な完了事項を簡潔に]

## 🗓️ 来週の見通し
[来週の重要日程]

## 💡 余裕があれば（任意）
[やっても良いが義務ではないもの]
```

→ 手順5へ進む

### 4. 平日ブリーフィングを生成する

```
# 朝のブリーフィング — YYYY年M月D日（曜）

## ⚠️ 引き継ぎ（未完了の重要タスク）
[直近ログから引き継ぎ事項を抽出]

## 📋 積み残し・すべきこと（全件）

### 🔴 今日中に必須
### 🟠 今週中
### 🟡 未解決の論点

## 🤖 今日使うべきスキル・エージェント
```

**重要**：タスクはTOP3に絞らず、積み残しをすべて列挙すること。

### 5. ブリーフィングをログファイルに保存する

今日の日付のログファイル `lab/logs/YYYY-MM-DD.md` に Write ツールで保存する。
ファイルが既に存在する場合は Read して内容を確認し、ブリーフィングセクションを追記する。

## 使用例

```
daily-start
daily-start "昨日の分析を途中で止めた。今日は続きから始めたい。"
daily-start "今日は午後から会議があるので、午前中に集中して作業する予定"
```
