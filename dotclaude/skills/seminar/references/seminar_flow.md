# ゼミ進行構造

## 【開会】司会による議題確認

司会が本日の議題を箇条書きで提示する。前回ゼミの未解決事項がある場合は冒頭で確認する。
議題提示後、**ユーザーに「この議題で進めてよいか」を確認してから開始する**。

## 【各議題の進行】

```
1. 司会が議題を提示し、最も関連の深いエージェント1名を指名候補として提案する
2. ユーザーが承認（または別のエージェントを指名）する
3. 司会が Agent ツールで指名されたエージェントを起動し、発言（opening statement）を得る
4. ユーザーが応答する（質問・反論・同意・別のエージェントへの指名など）
5. ユーザーの追加質問 → SendMessage で同じエージェントに転送、または別のエージェントを起動
6. 以下、ユーザーの応答→次の発言、を繰り返す
```

**重要**: ステップ3→4→5→4→5…の繰り返しにおいて、
**常にユーザーの発言を挟んでから次のエージェントが発言する**。
ユーザーの応答なしに2名以上のエージェントが連続で発言することは禁止。

## 【議題の区切り】

ユーザーが「次の議題に移ろう」「この議題はここまで」と言った場合に、
司会が暫定合意を1〜2文でまとめてから次の議題に進む。
形式：「確定事項 / 未解決事項 / 次のアクション」

## 【閉会】

ユーザーが閉会を指示した場合にのみ、以下をまとめる:

1. **アクションアイテム**（表形式：#・タスク・期限・担当）
2. **次回ゼミへの持ち越し論点**
3. **ユーザーへの未回答の問い一覧**（あれば）
