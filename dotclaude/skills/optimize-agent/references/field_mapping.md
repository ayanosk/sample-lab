# エージェント定義フィールドマッピング

## フロントマターフィールド

| フィールド | 必須 | 説明 |
|-----------|:----:|------|
| `name` | ○ | エージェントID（ハイフン区切り） |
| `description` | ○ | 1文の説明。「この agent を使うべき場面」を明示 |
| `tools` | ○ | エージェントが使えるツール |
| `model` | × | デフォルト省略（inherit）、軽量タスクは `sonnet` |

## tools 推奨設定

| エージェント種別 | 推奨 tools |
|-----------------|-----------|
| 読み取り専門（調査・分析） | `Read, Glob, Grep` |
| 分析＋出力（ライター・レポート） | `Read, Glob, Grep, Write` |
| 開発（エンジニア） | `Read, Glob, Grep, Write, Edit, Bash` |
| 管理（助手・メンテナンス） | `Read, Glob, Grep, Write, Bash` |

## system_prompt 最適化ガイドライン

1. **目標行数**: 30–60行
2. **構造**: 役割定義（3行）→ 専門分野（箇条書き）→ 行動指針（箇条書き）
3. **除去対象**:
   - CLAUDE.md や rules と重複する内容
   - 具体的なファイルパス（変わりやすい）
   - 過度に詳細な手順書（スキル側に持たせる）
4. **保持対象**:
   - このエージェント固有の専門性・視点
   - 応答スタイルの指示
   - 他エージェントとの差別化ポイント
