## 基本方針

- 常に `[応答言語]` で応答する
- まず対象プロジェクトの `README.md` と `CLAUDE.md` を読む
- 元データは `sources/` に置き、直接上書きしない
- 処理は `scripts/` で行い、結果は `outputs/` に出す
- 判断や気づきは `docs/` に残す
- Windows では Git Bash または WSL を優先して使う

## ディレクトリの考え方

- `lab/` は横断的な共通機能を置く
- `project_a/` や `project_b/` は個別プロジェクト
- `publications/` は外に出す資料や完成版

## 安全のためのルール

- 機密情報や API キーは Git に入れない
- `.env`、`.env.*`、`*.pem`、`*.key`、`*.p12`、`*.pfx`、`credentials/`、`secrets/` は読まない・コミットしない
- commit の前に差分を確認する
- **git push の前に必ず確認を取る**（自動で push しない）
- 初回セットアップでは確認モードを維持し、`Always allow` や `full-auto` を使わない
- 不安な操作は、まず確認を取ってから行う
- `dotclaude/` が残っている場合は、`.claude/` へのリネームを促す（README.md のセットアップ手順を参照）
