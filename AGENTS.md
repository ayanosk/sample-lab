# AGENTS.md テンプレート

Claude Code、Codex など複数の AI ツールで共有する基本ルールです。最初に自分用へ書き換えて使います。

> **初回セットアップ:** `dotclaude/` ディレクトリが存在する場合、まだセットアップが完了していません。
> [README.md](./README.md) の「セットアップ手順（Claude Code / Codex 向け）」を読んで実行してください。

## このリポジトリの目的

- `[例: 研究メモと分析コードを一元管理する]`
- `[例: 発表資料と論文下書きを整理する]`
- `[例: AI と安全に共同作業する]`

## 応答

- `[応答言語]` で応答する
- 作業前に、何をするかを短く伝える
- 作業後に、何ができたかを報告する

## 作業の進め方

- 対象プロジェクトの `README.md` を先に読む
- `sources/` は入力、`outputs/` は出力として扱う
- 分析や整形は `scripts/` を優先して再利用する
- 根拠やメモは `docs/` に残す
- Windows では、Claude Code は Git Bash / WSL、Codex は WSL ワークスペースを優先する

## 安全

- 機密情報は追加しない
- `.env`、`.env.*`、`*.pem`、`*.key`、`*.p12`、`*.pfx`、`credentials/`、`secrets/` は読まない・コミットしない
- 大きな生成物や一時ファイルは `.gitignore` で除外する
- 破壊的な操作は、必要性を説明してから行う
- 初回セットアップでは確認モードを維持し、`Always allow` や `full-auto` を使わない
- `git push`、依存追加、ネットワークアクセス、作業フォルダ外の読み書きは事前確認を取る

## ツール構成

このリポジトリは Claude Code と Codex の併用を前提にしています。
エージェント定義は `.claude/agents/`、スキルは `.claude/skills/` に配置されています。

Codex を使う場合は、README のセットアップ手順に従って `.claude/agents/` と `.claude/skills/` を `.agents/` にコピーしてください。
Codex は `AGENTS.md` を自動で読むため、共通ルールはここに集約します。
